 PROTOCOLLO TECNICO-OPERATIVO: DASHBOARD TENNIS ITALIA (VER. 5.1) PROTOCOLLO TECNICO-OPERATIVO: DASHBOARD TENNIS ITALIA (VER. 5.1) 1. Scopo del Progetto - Realizzare sito/dashboard per analisi statistica match tennis (ATP/WTA) orientato a identificare “Value Bet”. - Unisce dati TV (70 canali DTT + premium), quote bookmaker, KPI giocatori, meteo e H2H. - Output: segnali di puntata da “Gap Analysis” validata. 2. Architettura Modulare 2.1 Front-end - UI responsive (desktop/tablet/mobile). - Framework consigliato: React o Vue (oppure vanilla + HTMX se small). - Layout: - a sinistra: lista match + filtri - centro: dettaglio match (stats, H2H, superficie, forma) - destra: quote + gap indicator + betting-card popup - topbar: ricerca, selezione data, canali live. 2.2 Back-end - Node.js/Express oppure Python FastAPI. - API interne: - /api/matchs?date=... - /api/stats/player/{id} - /api/odds/{matchId} - /api/weather/{location} - /api/gap/{matchId} 2.3 Database - SQL (PostgreSQL) o NoSQL (Mongo). - Tabelle/collezioni: - canali (70 DTT + sugli OTT) + availability tennis - eventi (match, torneo, orario, canale, nationality) - atleti (KPI storici, superficie, classe) - h2h (storico matchup) - quote (bookmaker, tipo TB/OU/set, timestamp) - meteo (match esterno) - gap_analysis (scores, suggerimenti). 3. Integrazione Dati 3.1 Dati TV - Fonte: API EPG italiana + aggregazione modificata 70 canali DTT. - Mappatura obbligatoria “sport=tennis”. - Aggiornamento ogni 5m locale. 3.2 Quote Bookmaker - API validi (bet365, SNAI, Eurobet, Betfair, Pinnacle) o provider oddsfeed. - Feed continuo: - CF real time: tennis ATP/WTA + mercato exatti. - Polling 3-10 sec, WebSocket se possibile. - Tipologie: 1X2 (H2H), sets, game over/under. 3.3 Dati atletici - API ufficiali (ATP, WTA, Ultimate Tennis Stats se possibile). - KPI: - % prime palle - Ace/DF per set - BreakPoints convertiti/salvati - Tie-break win rate - WinRate superficie - Recent 10 match (forma). - Aggiornare ogni match e ogni fine torneo. 3.4 Meteo - API OpenWeatherMap / MeteoMatic. - Parametri: temperatura, umidità, vento, pressione. - Refresh 30 minuti + sul cambio match. - Influenza nel modello (palla life, durata, under/over). 4. Modulo Gap Analysis (Core) 4.1 Calcolo probabilità interna - Recavare probWin da modelli: - baseline: Elo rating + superficie + forma + h2h - weight: - superficie 30% - forma 20% - h2h 20% - meteo 10% - KPI fisici 20% - Converter market odds odd -&gt; implied probability 1/odd normalizzata. 4.2 Score - value_score = probWin - impliedProb - Azione: - &gt; 0.12 buy - 0.05-0.12 monitoraggio - &lt; 0 no bet - Plottare: - green/red bar - trend cambio quote (Tempo reale drop/flood) 4.3 Regole rischio - Kelly fraction ogni segnale. - staking consigliato (unità bankroll). - funzione “stop loss” basata su drawdown 5-10%. 5. UI / UX 5.1 Dashboard match - Filtri: - data, torneo, superficie, italiani, match in diretta, value &gt; soglia - Griglia: - orario, torneo, giocatore 1/2, canale, quote best, gap. - Stato: - ⚡ live, ⏳ pre-match, ✳ post. 5.2 Scheda Betting-Ready - Popup con: - riepilogo numerico + grafici. - comparativo quote bookies e miglior. - meteo + stato terreno. - campo canali (dove vedere). - segnale value bet + stake consigliato. 5.3 Sezione TV - Mappa canali (70 DTT + premium). - Mini-player / link streaming (ove legittimo). - Palinsesto orario tennis filtrabile. 5.4 Tracciamento e storico - Registro tipologie: - match, quota puntata, risultato, ROI. - Kpi betting personali: - profit/loss, ROI, strike rate. - Grafici: - equity curve, probabilità v. realizzato. 6. Requisiti funzionali e non-funzionali 6.1 Funzionali - feed realtime EPG + quote + meteo + stats. - calcolo value in &lt;1s. - navigazione mobile-first. - salva preferiti + alert price. - esporta CSV/e-mail report. 6.2 Non-funzionali - SLA 99,5 uptime. - query &lt;200ms su DB. - protezione API key + rate-limit 1000 req/min. - compliance GDPR + T&amp;C scommesse. 7. Sicurezza / Legalità - Disclaimer: strumento informativo, non bookie. - Non gestire denaro diretto. - Blockchain audit log (optional). - Protezione XSS/CSRF. - Rate limit API. - Audit log per tutte le modifiche. 8. Pianificazione Release 1. MVP1: - lista match, statistiche base, quote snapshot, UI iniziale. 2. MVP2: - Gap Engine + score + meteo + live update. 3. MVP3: - Betting card, storico, import/expor, alert. 4. Hardening: - API premium + monitoring + unit test + E2E.
